# Board Type — Habit Tracker (Phase 1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `type` discriminator to `Board` (`KANBAN` | `HABIT_TRACKER`), expose a type selector in the create-board modal, and stub the habit-tracker board detail view with a placeholder `<h1>` — without breaking any existing Kanban behavior.

**Architecture:** Prisma enum on `Board.type` (default `KANBAN`). `add_board_schema` gains a `type` field and `columns` becomes optional. `createBoardAction` branches on type to skip column creation for habit trackers. `prefetchBoard` refactored to return `{ dehydrated_state, board }` so the detail page can read `board.type` server-side. UI: create modal adds a Board Type select and only renders the columns block for Kanban; detail page renders current UI for Kanban or a stub for Habit Tracker.

**Tech Stack:** Next.js 16 (App Router), React 19, TypeScript 5, Prisma 7 (PostgreSQL), Zod, react-hook-form, TanStack React Query v5, shadcn/ui (Select), Tailwind v4.

**Spec:** `docs/superpowers/specs/2026-04-18-board-type-habit-tracker-design.md`

- [ ] **Step 4.2: Branch `createBoardAction` on board type**

In the same file, replace the entire body of `createBoardAction` (the `.action(async ({ parsedInput, ctx }) => { ... })` callback) with the following. The only functional change is guarding `columns.create` behind `type === "KANBAN"` and persisting `type` on the new board. Column order population still runs unconditionally but collapses to `[]` for habit trackers because `new_board.columns` is `[]`.

```ts
export const createBoardAction = authActionClient
	.schema(add_board_schema)
	.action(async ({ parsedInput, ctx }) => {
		const board = await prisma.$transaction(async (tx) => {
			/* Create the board with its columns (only for KANBAN type) */
			const new_board = await tx.board.create({
				data: {
					name: parsedInput.name,
					type: parsedInput.type,
					userId: ctx.userId,
					columns:
						parsedInput.type === "KANBAN" && parsedInput.columns
							? { create: parsedInput.columns.map((column) => ({ name: column.name, theme: column.theme })) }
							: undefined
				},
				include: {
					columns: { select: { id: true }, orderBy: { createdAt: "asc" } }
				}
			});

			/* Set the column order on the board (empty array for non-KANBAN) */
			const updated_board = await tx.board.update({
				where: { id: new_board.id },
				data: { columnOrder: new_board.columns.map((column) => column.id) },
				select: {
					id: true,
					name: true,
					type: true,
					columnOrder: true
				}
			});

			/* Append the new board to the user's board order */
			await tx.user.update({
				where: { id: ctx.userId },
				data: { boardOrder: { push: updated_board.id } }
			});

			return updated_board;
		});

		return board;
	});
```

export const prefetchBoard = async (board_id: string) => {
	const queryClient = getQueryClient();

	await queryClient.prefetchQuery({
		queryKey: [...CACHE_KEY_BOARD, board_id],

		staleTime: STALE_TIME

interface Props {
	params: {
		board_id: string;
	}

/**
 * DOCU: Board detail page. Renders Kanban UI or Habit Tracker stub depending on board.type. <br>

const TaskPage = async ({ params }: Props) => {
	const { board_id } = (await params) as { board_id: string };
	const { dehydrated_state, board } = await prefetchBoard(board_id);

	return (
		<HydrationBoundary state={dehydrated_state}>
			<Navbar />

			<MainContainer>
				{board?.type === "HABIT_TRACKER" ? (
					<div className="p-[24]">
						<h1 className="text-h-xl">Habit Tracker</h1>

						<EditTagsModal />
						<DeleteBoardModal />
						<ViewTaskModal />
						<EditTaskModal />
						<DeleteTaskModal />
					</div>
				)}
			</MainContainer>

	);
};

export default TaskPage;
